Summary:
[modules][reflection] conflicting type for __mbstate_t when combining import std,  in global module fragment, and -freflection

Product: gcc
Component: c++
Version: 16.1.1 (x86_64-pc-linux-gnu)
Target: x86_64-pc-linux-gnu

Description:
When including <cmath> (or <math.h>) in the global module fragment and importing the std module, compilation fails with a conflicting type error for __mbstate_t if and only if -freflection is enabled. The compiler fails to merge the identical declarations from the textually included header and the imported module metadata.

Without -freflection, the compilation succeeds.

Minimal Reproducer (test.ixx):

C++
module;
#include <cmath>
export module test;
import std;
Steps to Reproduce:

Bash
mkdir -p gcm.cache
g++ -std=c++26 -fmodules-ts -fmodule-only -c /usr/include/c++/16.1.1/bits/std.cc -o gcm.cache/std.gcm
g++ -c -std=c++26 -freflection -fmodules-ts test.ixx
Expected Results:
Clean compilation. The compiler should deduplicate/merge the __mbstate_t declarations successfully, exactly as it does when the -freflection flag is omitted.
